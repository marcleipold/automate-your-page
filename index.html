Language Ambiguity

Why write in Python instead of normal English?

  1. Natural languages are ambiguous ( open to more than one interpretation- a computer will run programs with the meaning we intended, thus we must kow exactly what our programs mean)
  2. Natural languages are very verbose (wordy- to say something with the level of precision needed for a computer to be able to follow it mechanically would require an awful lot of writing)

Our goal in writing a program is for it to be short, so it is less work to write them, and easier to read and understand them.

Grammar

Unlike a natural language, programming languages adhere to a strict grammatical structure. In English, even if something is said or written incorrectly, it can still be understood with the help of context or other cues. In programming, the code must match the language grammar exactly. The Python interpreter has no idea what to do with input that is not in the Python language.

  • Basic English Grammar Rules:
    • sentence--subject, verb, noun
    • subject--noun
    • object--noun
    • verb--eat
    • verb--like
    • noun--I
    • noun--Python
    • noun--cookies

If programming grammar is not followed, the interpreter will return a syntax error; this means the structure of the code is inconsistent with the rules of the programming language. Backus-Naur Form is used to describe a programming language in a simple and concise manner.

  • <non-terminal>-->replacement>

The "replacement" can be any sequence of zero, more-terminals, or more non-terminals. Terminals never appear on the left side of a rule. Once you get a terminal there is nothing else you can replace it with.

  • sentence--subject verb noun
  • sentence--noun verb object
  • sentence--I verb object
  • sentence--I like object
  • sentence--I like noun
  • sentence--I like Python

In the above example, the "I like Python" has been replaced with ALL terminals, therefore it can't be replaced anymore.

Python Expressions

An expression is something that has a value. Examples of expressions in Python:

  • 3
  • 1+1
  • 7*7*24*60

One of the rules of Python grammar for making expressions: Expression-->Expression Operator Expression. The Expression non-terminal on the left can be replaced by an Expression, followed by an Operator, followed by another Expression. It can have an Expression on both the left and right sides; this may look circular, and would be except for other rules for Expressions that don't include Expression on the right. This is an example of recursive definition (we'll see more in Unit 6). To have good recursive definition, we need two rules:

  1. A rule that defines something in terms of itself. (Expression-->Expression Operator Expression)
  2. A rule that defines that thing in terms of something else that we already know. (Expression-->Number)

Recursive definitions are a powerful idea in Computer Science. They allow us to define infinitely many things using a few simple rules. More on that in Unit 6... Here are some Python grammar rules for arithmetic expressions:

  • expression-->expression operator expression
  • expression-->number
  • operator-->+
  • operator-->*
  • number-->0,1,...

Here is an example of derivation using this grammar:

  • Expression-->Expression Operator Expression
    • -->Expression + Expression
    • -->Expression + Number
    • -->Expression + 1
    • -->Expression Operator Expression + 1
    • -->Number Operator Expression + 1
    • -->2 Operator Expression + 1
    • -->2 * Expression + 1
    • -->2 * Expression Operator Expression + 1
    • -->2 * Number Operator Expression + 1
    • -->2 * 3 Operator Expression + 1
    • -->2 * 3 * Expression + 1
    • -->2 * 3 * Number + 1
    • -->2 * 3 * 3 + 1

We only have to add one more rule to our expression grammar to be able to produce all the expressions we have used so far: expression-->(expression)

Unit 5: Variables and Strings

Basics of Variables and Strings

Variables use an assignment statement, such as:

  • name=expression
  • speed_of_light=299792458
  • speed_of_light refers to value-->299792458

So we can use variables to reference values/data, instead of using an actual value. Variables can...vary (crazy, right?) so we can update our expression. Also there is no hard fast rule, but typically variables shouldn't be Capitalized. It's important to remember that "=" is an assignment, it doesn't mean that the values are equal!

Strings are a sequence of characters surrounded by quotes. We can use single or double quotes. The advantage to double quotes is that you can use single quotes inside of doube quotes.

  • name="dianna"
  • name='dianna'

The + in strings is concatenation (a linking of things together in a series) of the two strings.

  • print "hello"+name
  • so as not to say "hellodianna", add space after "hello "
  • can't add a string and number together; however, we can multiply them (print "!"*3--> will give you 3 !'s)
